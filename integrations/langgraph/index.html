AIBrain + LangGraph
Persistent Memory for LangGraph Stateful Agents

AIBrain gives LangGraph graphs a checkpoint store backed by a local SQLite database. Pass checkpointer=store when compiling your graph and every state snapshot is persisted, searchable, and resumable across process restarts — no Redis, no Postgres required.

$ pip install aibrain

Quick Start

Initialize the checkpoint store and pass it to graph.compile(). LangGraph persistent state with zero infrastructure.

from aibrain.integrations.langgraph_adapter import AIBrainCheckpointStore
from langgraph.graph import StateGraph

# Initialize the checkpoint store
store = AIBrainCheckpointStore(user_id="workflow-1")

# Compile your graph with AIBrain as the checkpointer
graph = StateGraph(...)
compiled = graph.compile(checkpointer=store)

# Save and retrieve checkpoints directly
store.put(thread_id="t1", checkpoint_id="c1", data={"messages": [], "step": 3})
state = store.get(thread_id="t1")          # latest checkpoint
all_states = store.list(thread_id="t1")   # full history, newest first

Why AIBrain for LangGraph

Without AIBrain, LangGraph workflow state lives in memory and vanishes on restart. Every graph execution starts from scratch. AIBrain adds persistent memory for LangGraph that survives process restarts, stores full graph state as searchable memories, and lets you resume any thread from its last checkpoint.

Native LangGraph Checkpointer

Implements the checkpointer interface so graph.compile(checkpointer=store) works out of the box. No adapter glue code needed.

Full State Persistence

Each checkpoint stores the complete serialised graph state dict, including messages, step counters, and all custom fields your graph uses.

Thread-Scoped History

store.list(thread_id=...) returns all checkpoints for a thread ordered newest-first — full replay capability for debugging and resumption.

Zero Infrastructure

No Redis, no Postgres. Checkpoints are stored in AIBrain's local SQLite database. Works offline, works on any machine.

Survives Restarts and Deploys

Graph state persists across process restarts, deployments, and machine reboots. Pick up exactly where you left off.

LangGraph Stateful Agent Memory

Works for all LangGraph graph types — from simple chatbots to complex multi-agent orchestration pipelines with branching state.
