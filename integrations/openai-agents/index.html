AIBrain + OpenAI Agents SDK
Persistent Memory for OpenAI Agents

AIBrain gives OpenAI Agents SDK agents two native memory tools — memory_search_tool and memory_store_tool — that agents call autonomously. The agent decides when to remember and when to recall. Knowledge compounds across every run without any manual wiring.

$ pip install aibrain

Quick Start

Register the memory tools on your agent, pass AIBrainContext to the runner. The agent handles the rest.

from aibrain.integrations.openai_agents_adapter import (
    AIBrainContext,
    memory_search_tool,
    memory_store_tool,
)
from agents import Agent, Runner

agent = Agent(
    name="assistant",
    instructions="You are a helpful assistant with persistent memory.",
    tools=[memory_search_tool, memory_store_tool],
)

ctx = AIBrainContext(user_id="openai-alice", k=5)
result = Runner.run_sync(agent, "What do you remember about me?", context=ctx)
print(result.final_output)

# Store a fact explicitly
ctx.save("Alice prefers Python over JavaScript.")
# Next session — same user_id, agent recalls it automatically

Why AIBrain for OpenAI Agents

Without AIBrain, OpenAI Agents SDK agents have no memory between runs. Each Runner.run_sync() call starts cold. AIBrain adds OpenAI Agents persistent memory via tools the agent controls natively — it stores facts as it learns them and searches its memory before responding, so knowledge compounds across every run.

Agent-Native Tools

memory_search_tool and memory_store_tool are registered as @function_tool when the SDK is available. The agent calls them like any other tool — no extra wiring.

Typed RunContext Integration

AIBrainContext is a dataclass that passes cleanly through the SDK's dependency injection pattern. No global state, no thread-local hacks.

Deduplication Built In

memory_store_tool returns "Memory already known" when a fact is already stored. The memory store stays clean — no duplicates ever accumulate.

Direct Context API

ctx.search(), ctx.save(), and ctx.recall_as_str() for direct use in tool or orchestration code outside the agent loop.

Multi-User and Multi-Agent Safe

Memory is scoped per user_id and agent_id. Safe for multi-user and multi-agent deployments with zero cross-contamination.

Zero Infrastructure

Local SQLite. No external services. Works offline, works anywhere. Same machine or containerized — no changes needed.
