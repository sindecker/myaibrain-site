AIBrain + CrewAI
Persistent Memory Between CrewAI Runs

AIBrain gives CrewAI crews a persistent memory provider that survives between runs. Without AIBrain, every kickoff() starts fresh — research done in run 1 is gone by run 2. With AIBrain, knowledge compounds: every fact a crew discovers is stored and retrieved semantically on every subsequent run.

$ pip install aibrain

Quick Start

Initialize AIBrainCrewMemory and pass it as memory_provider. CrewAI persistent memory with one extra line.

from aibrain.integrations.crewai_adapter import AIBrainCrewMemory
from crewai import Crew, Agent, Task

# Initialize the memory provider
memory = AIBrainCrewMemory(user_id="crew-research")

# Store findings directly
memory.save("The target company was founded in 2019 in Berlin.")

# Semantic search over stored knowledge
results = memory.search("When was the company founded?", limit=3)
for r in results:
    print(r["memory"], r["score"])

# Wire into a crew
researcher = Agent(role="Researcher", goal="Find company info", memory=True)
crew = Crew(
    agents=[researcher],
    tasks=[Task(description="Research the company", agent=researcher)],
    memory_provider=memory,
)
crew.kickoff()

Why AIBrain for CrewAI

Without AIBrain, CrewAI crews start fresh on every kickoff(). Research done in run 1 is gone by run 2. AIBrain adds CrewAI persistent memory between runs — every fact a crew discovers is stored and retrieved semantically, so your crew accumulates expertise instead of repeating the same work.

Full memory_provider Interface

save, search, and reset match the CrewAI memory provider contract exactly. Pass memory_provider=memory with no other changes to your crew.

Per-Agent Memory Scoping

agent_id parameter on save() and search() lets individual agents maintain isolated memory lanes within the same crew.

Semantic Search with Relevance Scores

Results include id, memory, score, and reason fields. Crews can rank and filter retrieved knowledge for precision.

Automatic Fact Extraction

infer=True on save — AIBrain extracts structured facts from free-text, making retrieval more precise than storing raw strings.

Knowledge That Compounds

Every fact discovered in run 1 is available in run 2. Crews improve over time without you writing any consolidation logic.

Clean Soft-Delete Semantics

Memories are soft-deleted — removed from retrieval but retained for audit. Historical knowledge is never permanently lost.
