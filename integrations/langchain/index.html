AIBrain + LangChain
Persistent Memory for LangChain Agents

AIBrain gives your LangChain chains and agents a persistent memory layer backed by a local vector database. Without AIBrain, every LangChain session starts blank. With AIBrain, knowledge compounds — the longer you run, the smarter your agent gets.

$ pip install aibrain[langchain]

Quick Start

Drop AIBrainMemory in as a BaseMemory replacement. Two lines changed, full LangChain persistent memory across sessions.

from aibrain.integrations.langchain_memory import AIBrainMemory
from langchain.chains import ConversationChain
from langchain_openai import ChatOpenAI

# session_id scopes memory to this user — persists across restarts
memory = AIBrainMemory(session_id="user-123", top_k=10)
chain = ConversationChain(llm=ChatOpenAI(), memory=memory)

chain.predict(input="My favourite colour is blue")
# Next session, same session_id:
chain.predict(input="What is my favourite colour?")
# → "Your favourite colour is blue."

With LCEL / RunnableWithMessageHistory

from aibrain.integrations.langchain_memory import AIBrainChatMessageHistory
from langchain_core.runnables.history import RunnableWithMessageHistory

runnable_with_history = RunnableWithMessageHistory(
    runnable,
    lambda session_id: AIBrainChatMessageHistory(session_id=session_id),
)

Why AIBrain for LangChain

Without AIBrain, LangChain agents forget everything between sessions. Every conversation starts blank. AIBrain adds a LangChain persistent memory layer that compounds across sessions — session context from previous conversations is retrieved semantically, not just replayed in full.

Drop-in BaseMemory Replacement

AIBrainMemory is a first-class BaseMemory subclass. Swap it into any existing ConversationChain with no other changes.

LCEL / LangGraph Compatible

AIBrainChatMessageHistory implements BaseChatMessageHistory for RunnableWithMessageHistory pipelines and LangGraph state graphs.

Semantic Retrieval Per Turn

On every load_memory_variables call, AIBrain searches for the most relevant prior context — not just the last N messages. Context stays lean and accurate.

Session-Scoped Isolation

Memories are namespaced by session_id so multi-user deployments stay clean. No cross-contamination between users or agents.

Zero Infrastructure

No Redis, no Postgres, no broker. Memories are stored in AIBrain's local SQLite database — works on any machine, no external dependencies.

LangChain Memory That Persists

Soft-delete semantics retain memories for audit while keeping retrieval clean. Full change history tracking on every memory entry.
